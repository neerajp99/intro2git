A beginners' intro to git

Simple guide to learn git in less than 30 minutes

setting a repository

creates a new Git repository
git init

executing git init creates a .git subdirectory in the project root

check the status

checks the current state of repository
git status
output
$ git status
# On branch master
nothing to commit (working directory clean)

checkout a repository

create a working copy of a local repository by running the command

git clone /path/to/repository

when using a remote server, your command will be

git clone username@host:/path/to/repository

git branch

list all of the branches in your repository

git branch

 create a new branch list all of the branches in your repository
git branch <branch>

delete the specified branch
git branch -d <branch>

rename the current branch to <branch>
git branch -m <branch>

git checkout

checking out files, checking out commits, and checking out branches

return to the master branch
git checkout master

to check the previous version
git checkout <commit > <file>

update all files in the working directory to match the specified commit
git checkout <commit>

to show commit logs

display the entire commit history
git log

limit the number of lines by <limit>
git log -n <limit>

condense each commit to a single line
git log --oneline

show only commits that occur between <since> and <until>
git log --stat

only display commits that include the specified file
git log <file>

graph flag draw a text based graph of the commits on the left hand side of the commit messages
git log --graph

updating remote refs

The git remote command lets you create, view,
and delete connections to other repositories.

list the remote connections you have to other repositories
git remote

to create a new connection to a remote repository
git remote -v

to remove the connection to the remote repository called <name>
git remote rm <name>

rename a remote connection from <old-name> to <new-name>
git remote rename <old-name> <new-name>

pushing remote refs


push the specified branch to <remote>
git push <remote> <branch>

push the specified branch to <remote> but force the push even if it results in a non-fast-forward merge
git push --force

Push all of your local branches to the specified remote
git push --all

forward-port local commits

rebasing really is just moving a branch from one commit to another

Rebase the current branch onto <base>
git rebase <base>

stashing

to record the current state of the working directory
git stash

the modification stashed away by git stashcan be listed
git stash list

reset current HEAD to the specified state

remove the specified file from the staging area, but leave the working directory unchanged
git reset <file>

reset the staging area to match the most recent commit
git reset

reset the staging area and the working directory
git reset --hard

Move the current branch tip backward to <commit>
git reset <commit>
